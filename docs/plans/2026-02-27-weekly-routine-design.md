# 设计文档: 结构化周训练计划 (Weekly Routine)

## 1. 目标
将原有的“扁平动作列表”升级为“以周为维度的结构化训练安排”。用户可以为星期一到星期日分配不同的训练动作，或是将其标记为“休息日”。完成当天的任务后，对应的星期会被“点亮”（标记为已完成）。

## 2. 数据结构 (JSON 方案)
在 `PlanEntity` 中，将原本简单的 `exercisesJson` 升级为如下结构：

```json
[
  { "dayOfWeek": 1, "isRest": false, "exercises": ["benchpress", "pushup"] },
  { "dayOfWeek": 2, "isRest": true, "exercises": [] },
  ...
  { "dayOfWeek": 7, "isRest": false, "exercises": ["squat"] }
]
```
*(注：dayOfWeek: 1 代表星期一，7 代表星期日)*

同时，由于“点亮”状态（即本周完成了哪些天）会随着时间推进（下一周会重置），所以：
- **静态安排** 存在 `PlanEntity` 中。
- **动态进度（是否点亮）** 不存在 Plan 里，而是通过查询当前周的 `SetEntity`（训练记录）实时计算得出。只要今天有完成对应计划内的动作，该天即被点亮。

## 3. UI/UX 设计
**Plans Tab (计划首页)**
- **顶部进度条**：7 个圆形的小徽章 (M, T, W, T, F, S, S)。
  - 灰色：未来/未完成
  - 主题色（高亮）：今天或已完成的训练日
  - 绿色/空心：休息日
- **今日任务卡片**：
  - 如果今天是训练日：显示今天要做的动作列表，以及一个醒目的“开始今天的训练”按钮。
  - 如果今天是休息日：显示“今天是休息日，好好放松一下吧”。
- **周计划预览列表**：列出周一到周日每天的安排概览（方便用户查看和修改）。

**PlanSession Screen (训练执行页)**
- 点击“开始今日训练”后进入。
- 仅加载当前星期的动作列表。
- 完成动作后点击打卡。

## 4. 迁移与兼容性
- 破坏性更新：由于 JSON 结构完全改变，Room 数据库需要清除重建（或清空旧的测试 Plan），以适应新的数据模型。
